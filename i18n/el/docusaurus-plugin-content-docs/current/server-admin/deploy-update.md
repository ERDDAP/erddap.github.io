# Ενημέρωση
Πώς να Κάνετε μια Ενημέρωση μιας Υπάρχουσας ERDDAP™ στον εξυπηρετητή σας

## Μεταβολές{#changes} 
1. Κάντε τις αλλαγές που αναφέρονται στο [Μεταβολές](/changes) στο τμήμα με τίτλο " Πράγματα ERDDAP™ Οι διαχειριστές πρέπει να γνωρίζουν και να κάνουν" για όλους τους ERDDAP™ εκδόσεις από την έκδοση που χρησιμοποιούσατε.
     
##  Java  {#java} 
2. Εάν αναβαθμίζετε από ERDDAP™ έκδοση 2.18 ή κάτω, θα πρέπει να αλλάξετε σε Java 25 (ή νεότερη) και το σχετικό Tomcat 10. Δείτε την τακτική ERDDAP™ οδηγίες εγκατάστασης για [ Java ](/docs/server-admin/deploy-install#java) και [Τομκάτ](/docs/server-admin/deploy-install#tomcat) . Θα πρέπει επίσης να αντιγράψετε _tomcat_/content/erddap κατάλογος από την παλιά εγκατάσταση Tomcat σας στη νέα εγκατάσταση Tomcat.

## Λήψη{#download} 
3. Λήψη [erddap.war (στα Αγγλικά).](https://github.com/ERDDAP/erddap/releases/download/v2.30.0/ERDDAP-2.30.0.war) _tomcat_/webapps .
     (έκδοση 2.30.0, 706.939.130 bytes, MD5=CDC4B3D82A20B33A6623B85312F6DC21, με ημερομηνία 2026-04-06) 
     
## μηνύματα.xml{#messagesxml} 
4. 
    * Συχνές: Εάν αναβαθμίζετε από ERDDAP™ έκδοση 1.46 (ή πάνω) και απλά χρησιμοποιείτε τα τυποποιημένα μηνύματα, το νέο πρότυπο μηνύματα.xml θα εγκατασταθεί αυτόματα (μεταξύ των αρχείων . τάξης μέσω erddap. πόλεμος) .
         
    * Σπάνιες: Εάν αναβαθμίζετε από ERDDAP™ έκδοση 1.44 (ή κάτω) ,
ΠΡΕΠΕΙ ΝΑ διαγράψετε το παλιό αρχείο μηνυμάτων.xml:
         _tomcat_/content/erddap /μήνυμα.xml .
Τα νέα πρότυπα μηνύματα.xml θα εγκατασταθούν αυτόματα (μεταξύ των αρχείων . τάξης μέσω erddap. πόλεμος) .
         
    * Σπάνιες: Αν κάνετε πάντα αλλαγές στο τυπικό αρχείο μηνυμάτων.xml (σε θέση) ,
πρέπει να κάνετε αυτές τις αλλαγές στο νέο αρχείο μηνυμάτων.xml (που είναι
WEB-INF/classes/gov/noaa/pfel/erddap/util/messages.xml μετά την αποσυμπίεση του erddap.war από τον Tomcat).
         
    * Σπάνιες: Αν διατηρείτε ένα προσαρμοσμένο αρχείο μηνυμάτων.xml _tomcat_/content/erddap /,
Πρέπει να καταλάβεις. (μέσω diff) ποιες αλλαγές έχουν γίνει στα προεπιλεγμένα μηνύματα.xml (που βρίσκονται στο νέο erddap. πόλεμο ως
WEB-INF/classes/gov/noaa/pfel/erddap/util/messages.xml) και τροποποιούν τα προσαρμοσμένα μηνύματά σας.xml αρχείο αναλόγως.
         
## Εγκατάσταση{#install} 
5. Εγκατάσταση του νέου ERDDAP™ στο Tomcat:
\\* Μην χρησιμοποιήσετε Tomcat Manager. Αργά ή γρήγορα θα υπάρξουν προβλήματα μνήμης PermGen. Καλύτερα να κλείσουμε και να ξεκινήσουμε το Tomcat.
\\* Αντικατάσταση αναφορών στο _tomcat_ παρακάτω με τον πραγματικό κατάλογο Tomcat στον υπολογιστή σας.
     
### Linux και Macs{#linux-and-macs} 
1. Κλείσιμο Tomcat: Από μια γραμμή εντολών, χρήση: _tomcat_/bin/shutdown.sh
Και χρήση ps -ef | grep tomcat για να δείτε αν/όταν η διαδικασία έχει σταματήσει. (Μπορεί να πάρει ένα ή δύο λεπτά.) 
2. Αφαιρέστε το αποσυμπιεσμένο ERDDAP™ εγκατάσταση: Σε _tomcat_/webapps, χρήση
rm - rf erddap
3. Διαγράψτε το παλιό erddap. αρχείο πολέμου: Σε _tomcat_/webapps, χρήση rm erddap. πόλεμος
4. Ελήφθη το νέο erddap. αρχείο πολέμου από τον προσωρινό κατάλογο σε _tomcat_/webapps
5. Επανεκκίνηση Tomcat και ERDDAP : χρήση _tomcat_/bin/startup.sh
6. Προβολή ERDDAP™ στον περιηγητή σας για να ελέγξετε ότι η επανεκκίνηση πέτυχε.
     (Συχνά, πρέπει να προσπαθήσεις μερικές φορές και να περιμένεις ένα λεπτό πριν δεις ERDDAP™ .)   
             
### Windows{#windows} 
1. Κλείσιμο Tomcat: Από μια γραμμή εντολών, χρήση: _tomcat_\\bin\\ shutdown.bat 
2. Αφαιρέστε το αποσυμπιεσμένο ERDDAP™ εγκατάσταση: Σε _tomcat_/webapps, χρήση
Δελ /S/Q erddap
3. Διαγράψτε το παλιό erddap. αρχείο πολέμου: Σε _tomcat_\\webapps, χρήση del erddap. πόλεμος
4. Ελήφθη το νέο erddap. αρχείο πολέμου από τον προσωρινό κατάλογο στο _tomcat_\\webapps
5. Επανεκκίνηση Tomcat και ERDDAP : χρήση _tomcat_\binbin\\startup.bat
6. Προβολή ERDDAP™ στον περιηγητή σας για να ελέγξετε ότι η επανεκκίνηση πέτυχε.
     (Συχνά, πρέπει να προσπαθήσεις μερικές φορές και να περιμένεις ένα λεπτό πριν δεις ERDDAP™ .)
