Εκτέλεση ERDDAP™ στο σύννεφο έχει γίνει ένα καυτό θέμα. Θα πρέπει να σημειώσω ότι ERDDAP™ πάντα τρέχει στο cloud, μόνο τις περισσότερες φορές δεν σε έναν εξυπηρετητή που παρέχεται από έναν εμπορικό πάροχο cloud, και το κύριο εμπόδιο για την εκτέλεση ERDDAP™ σε έναν εμπορικό πάροχο cloud είναι αν χρησιμοποιείτε S3 αποθήκευσης, η οποία δεν επιτρέπει τη συνήθη πρόσβαση μπλοκ Linux. Αν είστε πρόθυμοι να πληρώσετε περισσότερα για να χρησιμοποιήσετε τις επιλογές πρόσβασης μπλοκ που παρέχονται από τον εμπορικό πάροχο cloud σας, από το να τρέχει σε ένα εμπορικό διακομιστή cloud είναι βασικά το ίδιο με το τρέξιμο με δικό σας εξοπλισμό, εκτός φυσικά το κόστος.

Τούτου λεχθέντος, την 1η Δεκ. 2025 έγραψα μια ανάρτηση “Rclone and S3” και αυτή είναι μια συνέχεια. Σε αυτό το e-mail ανάρτησα τα GOES17 swattes και έλεγξα ένα αρχείο, αλλά δεν το πήρε σε όλη τη διαδρομή στο ERDDAP™ Για να δούμε ότι όλα λειτουργούν ομαλά. Και ναι παιδιά, μπορείτε να δοκιμάσετε αυτό στο σπίτι και δεν χρειάζεται να συμβουλευτείτε ένα δικηγόρο ή ιατρικό σύμβουλο, όλα θα πρέπει να είναι ασφαλή. Εδώ ανεβάζω το NCDC OI sst avhrr v2.1 που είναι σε AWS, να συσταθεί σε ERDDAP™ και τα αποτελέσματα.

- Βήμα 3: τοποθετήστε την αποθήκευση S3 στο σημείο προσάρτησης

Άδειες, άδειες, άδειες, άδειες... (Με συγνώμη στον Στιβ Μπάλμερ, αν ξέρεις ότι...) ,

Η προσάρτηση πρέπει να γίνει έτσι ώστε οποιοσδήποτε χρήστης τρέχει το γάτο σας, συνήθως ο χρήστης “τομκάτος”, να μπορεί να έχει πρόσβαση στα δεδομένα. «Rclone» τοποθετεί το σύνολο δεδομένων με τον ιδιοκτήτη και την ομάδα του χρήστη που εκτελεί την εντολή προσάρτησης και θέλει να αποθηκεύσει πληροφορίες στον αρχικό κατάλογο του χρήστη (Αυτό πιθανώς μετριάζεται αν το ρυθμίσετε ως διαδικασία επιπέδου συστήματος - δείτε παρακάτω) . Έτσι, αν μπορείτε, εκτελέστε την εντολή προσάρτησης ως 'tomcat', αλλά αν σαν εμάς το tomcat σας δεν έχει έναν κατάλογο στο σπίτι θα πρέπει να εκτελέσετε την εντολή προσάρτησης ως διαφορετικό χρήστη. Για να γίνει αυτό επεξεργαστείτε το φυτίλι. αρχείο conf:

1. sudo vi /etc/fuse.conf (στα Αγγλικά).

2. Αποσύνδεση ή προσθήκη:

_επιτρέπει_άλλο

3. Αποθήκευση και έξοδο.

rclone -vv mount oi sst :noaa-cdr-sea-surface-temp-optimum-interpolation-pds/data/v2.1/avhrrr /mnt/oi sst \\

 <fileNameRegex> oi sst Εντάξει. .nc  </fileNameRegex> 

Συχνά έχω πει ότι θεωρώ ότι η ρέγκεξ είναι ένα από τα μυστήρια της ζωής, και μπορεί να υπάρχουν καλύτεροι τρόποι για να γίνει η ρέγκεξ. Αλλά αυτό έπιασε.

3. Η κατηγορία ioos δεν είχε οριστεί, πρόσθεσα αυτά.

Αν παίξετε με το αποτέλεσμα, σημειώστε πρώτα ότι οι ρυθμίσεις rclone είναι μια πρώτη εικασία, και θα πρέπει να δοκιμαστεί για βελτιστοποίηση. Ο Jonathan Sherman από την ομάδα μας το έχει εξετάσει και μπορεί να μιλάει γι' αυτό κατά τη διάρκεια της ομιλίας του στη συνάντηση της IOOS DMAC. Θα καλύπτει επίσης πολύ περισσότερα θέματα που σχετίζονται με τη δημιουργία στην πλατφόρμα Google Cloud, όπως το πώς να ενορχηστρώσει τη ρύθμιση του VM, τη δημιουργία του κουβά S3 για να έχουν ένα ιεραρχικό χώρο όνομα που στο GCP είναι πιο γρήγορα και μόνο λίγο πιο ακριβά, και αν εκτελέσετε την επεξεργασία σενάρια για την ενημέρωση των δεδομένων που εξυπηρετούνται από το ERDDAP™ Πώς να τα στήσεις αυτά. Αν αυτό το θέμα σας ενδιαφέρει σας ενθαρρύνω να ακούσετε την ομιλία του. Η ERDDAP™ είναι σε λειτουργία και λειτουργεί, απλά δεν είναι προσβάσιμο προς το παρόν από το εξωτερικό NMFS δίκτυο.

Δεύτερον, αυτό δεν είναι ένα AWS VM τοποθέτηση ενός AWS S3 κουβά, αυτό είναι ένας από τους διακομιστές μας και η πίπα μας αυτές τις μέρες είναι εντελώς κορεσμένη, έτσι θα περιμένατε η προηγούμενη εγκατάσταση να είναι ταχύτερη από ό, τι έχω κάνει (Λοιπόν η πίπα μας δεν είναι πολύ μεγάλη - ευχαριστώ NMFS - αλλά είμαστε ποτέ κορεσμένοι - η ζήτηση για δεδομένα ήταν εκπληκτική) .

Επιτέλους ίσως αναρωτιέστε - θέλω να κυλήσει το δικό μου, από πού μπορώ να ξεκινήσω εκτός από αυτό; Έχω βρει ένα πράγμα LLMs είναι καλή σε πληροφορίες που είναι πολύ γνωστές και καλά τεκμηριωμένες, και οι AI που έχω ελέγξει (Πάνε όλα μου τα δώρα&#33;) όλοι γνωρίζουν rclone και AWS και GCP αρκετά καλά, και μπορεί να κάνει το μεγαλύτερο μέρος της εγκατάστασης για σας. Στην πραγματικότητα έψαχνα για ένα σύνολο δεδομένων που θα ήταν καλό να demo, και ένα AI μου έδωσε αρκετές προτάσεις και δημιούργησε τα περισσότερα από ό, τι είναι πάνω, αν και έκανα κάποιες επεξεργασίες για τη δική μου εγκατάσταση.
