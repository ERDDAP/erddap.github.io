#  `ディスプレイ インフォメーション` そして、 `ディスプレイ属性` ニュース

## コンテンツ
この機能を使用すると、データセットページで選択したグローバル属性を表示できます。 `インフォメーション` 行。

## 使用法 指示
これらのタグは、使用できません `サックスパーサー` お問い合わせ それらを有効にして使用するために、次の手順に従ってください。

1.  **SAXパーサーを有効にする** : : :
次の行をあなたのところに追加します `セットアップ。xml` ファイル:
   ```xml
   <useSaxParser>true</useSaxParser>
   ```

2.  **タグを追加 ` datasets.xml ` ** : : :
お問い合わせ ` datasets.xml ` ファイルには、2つのトップレベルのタグが含まれます。
   ```xml
   <displayInfo></displayInfo>
   <displayAttribute></displayAttribute>
   ```

3.  **デフォルト行動** : : :
   - これらのタグが追加されていない場合、または空の左 ` datasets.xml ` file では、デフォルト値は次のようになります。
     -  `ディスプレイ インフォメーション` : : : `概要、免許証` 
     -  `ディスプレイ属性` : : : `概要、ライセンス` 

4.  **一貫性を確保** : : :
両方のコンマ区切り値の数 `ディスプレイ インフォメーション` そして、 `ディスプレイ属性` タグは同じでなければなりません。

## 仕組み
- ザ・オブ・ザ・ `ディスプレイ属性` tag はグローバル属性を指定します (定義は内部)&lt; ` addAttributes ` データセットごとに表示する&gt;タグ)。
- 対応する値は、 `ディスプレイ インフォメーション` タグはラベルとして表示されます `インフォメーション` UIの行。
- 表示されたラベルの上にユーザがカーソルを合わせると、ツールチップが現れ、グローバル属性の値を表示します。

## 事例紹介
```xml
<displayInfo>Display1,Display2</displayInfo>
<displayAttribute>att1,att2</displayAttribute>
```

### データセット グローバル属性例:
```xml
<att name="att1">This is att1</att>
<att name="att2">This is att2</att>
```

### UIの行動:
- キーワード `ディスプレイ1` そして、 `ディスプレイ2` に表示されます。 `インフォメーション` UI の行。
- hovered の場合、ツールチップは対応する属性値を表示します。
  -  `ディスプレイ1` : ツールチップ表示 _att1_
  -  `ディスプレイ2` : ツールチップショー _att2_

## インフォメーション
- 指定した属性名を必ず確認 `ディスプレイ属性` タグは、データセットで定義されたグローバル属性にマッチします。
- 誤りや欠落した属性はエラーメッセージを記録します。

これらの手順に従って、カスタマイズできます `インフォメーション` 関連するグローバル属性を対応するツールチップで表示するために、データセットページの行。
