# Suche und Anzeige von Videos in ERDDAP™ 

Dieses Dokument zeigt, wie man Videos in ERDDAP™ , sowie
Erstellen eines Datensatzes, der die Suche der Video-Metadaten ermöglicht
finden Sie die Videos, die die ausgewählten Suchkriterien erfüllen.

Die wesentlichen Komponenten, die für diese Einrichtung erforderlich sind, sind zunächst Videos, die
sind in einem Format, das in einem Webbrowser angezeigt werden kann ( ERDDAP™ selbst
das Video nicht angezeigt, es nur einen Link liefert), und zweitens, ein csv
Datei (oder ein ähnliches Tabellenformat wie Parkett) dass die
für die Videos relevante Metadaten.

Ein entsprechendes Videoformat zu haben, ist der Schlüssel. Im Beispiel unterhalb der
Videos waren ursprünglich ein altes Fenster-Format, dass sogar Windows-Browser
konnte nicht aus der Box angezeigt werden, und erforderte ein Plugin hinzufügen. Alter
Formate sind oft sehr große Dateien. Die Handbrake Anwendung kann
verwendet, um die Videos zu konvertieren, was getan wurde.

## Einrichtung

Der erste Schritt besteht darin, einen Datensatz "EDDTableFromFileNames" so zu erstellen, dass
es gibt eine URL, die jedem Video zugeordnet ist, die durch
die ERDDAP™ "Dateien" Befehl. Während ein Beispiel dieses Datensatzes gegeben ist
unten, der beste Weg, um diese xml für Ihre Videos zu generieren ist zu verwenden
Datensätze generieren Xml.sh und zeigen es in das Verzeichnis mit den Videos (in
die beste Möglichkeit, neue Datensätze hinzuzufügen ERDDAP™ ist zu verwenden
Datensätze generieren Xml.sh, die Ihnen den größten Teil des Weges dorthin bringt, und
dann das für das Endergebnis bearbeiten).

Der zweite Schritt ist die Schaffung eines EDDTableFromAsciiFiles (oder ähnliches) 
Datensatz, der die Metadaten enthält, die im Video verwendet werden sollen
Suche. Ein Beispiel ist unten angegeben. Für diese Arbeit muss man
Spalte, die einen Link zum Video enthält, im Beispiel ist es die
Variable “videoFile”. Diese Variable kann nur die volle URL enthalten
erstellt im ersten Schritt, aber dann ist die Datei nicht tragbar, wie wenn
Sie wollen auf einer Maschine testen und dann tatsächlich auf einer anderen dienen
Maschine.

Sie können die Datei tragbare machen, indem Sie den Eintrag in “videoFile” zu sein
der relative Pfad zum Video, im Beispiel unterhalb eines solchen Pfades ist:

2010_Piggy_Bank_audio_stripped/0001_20100627_144631.mp4

Dann können Sie das Attribut „fileAccessBaseUrl“ hinzufügen, das angibt, dass
URL auf den Pfad in der Spalte. Im Beispiel:

```
<att name="fileAccessBaseUrl"><https://coastwatch.pfeg.noaa.gov/erddap/files/fed_HAGE_VisualSurvey/></att>
```

Wenn Sie dann Server wechseln, müssen Sie nur dieses Attribut in ändern
die xml Schnipsel, Sie müssen die csv-Datei nicht bearbeiten.
