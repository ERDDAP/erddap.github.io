#  ERDDAP™ Процес випуску
* Переконайтеся, що файли порівняння зображень доступні (це може означати біг `mvn перевірка` , якщо ви хочете прискорити обмеження, щоб просто група ImageComparison хоча зауважити, що все ще вимагає тестування Jetty) 
* Оновлення залежності
```
mvn versions:display-dependency-updates   // (displays updates)
mvn versions:use-latest-versions  // (updates dependencies, though sometimes we don’t want to do all of them)
mvn versions:update-properties // (updates versions in the property block)
```
* Оновлення плагінів
```
mvn versions:display-plugin-updates // (displays updates, need to manually update)
```
* Тести, щоб переконатися, що оновлення залежності не порушили нічого для всіх основних конфігурацій (описи даних, зокрема, якщо будь-які інші значні налаштування, а також) й Зверніть увагу, що зовнішній тестовий люкс може бути дуже флакитним. Випробувано дуже довго.
```
mvn verify
mvn verify -P external
mvn verify -P slowAWS
```
* Зареєструватися `python переклад/translate.py` щоб оновити переклади, якщо необхідно.
* Розробка EDStatic.java Режим на false, змінити номер версії та вказати дату релізу.
* До збірки.
```
mvn clean
mvn compile
mvn package
```
## Канар
Відправте файл War для розподілу на сервері Coastwatch або іншому сервері, який використовує більшість типів даних і отримує багато трафіку.
Ми хочемо, щоб знайти помилки перед поширенням збірки.

Включіть повідомлення при розповіді про новий реліз.

Стандартна процедура:
* Завантажте файл .war до берегового годинника \\[ доматор \\] / контент/робота/
* Як користувач=tomcat:
  * У \\[ доматор \\] Українська
./shutdown.sh //use "ps -fu tomcat" для забезпечення його зупинки
  * У \\[ доматор \\] Веб-додатки / :
rm -rf erddap
rm erddap. війни
з ../content/erddap/erddap2.22.war erddap.war/або будь-який номер
  * У \\[ доматор \\] Українська
./startup.sh
  * Після ERDDAP повернула веб-сторінку, в \\[ доматор \\] Веб-додатки / :
хрп -Р ерддап ерддап
chmod -R g+rw erddap
Хмод -Р o-rwx erddap
