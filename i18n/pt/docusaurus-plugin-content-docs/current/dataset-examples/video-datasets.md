# Procurando e exibindo vídeos em ERDDAP™ 

Este documento mostra como exibir vídeos em ERDDAP™ , e
criar um conjunto de dados que fornece para pesquisar os metadados de vídeo para
encontrar os vídeos que atendem aos critérios de pesquisa selecionados.

Os componentes essenciais necessários para configurar isso são os primeiros, vídeos que
estão em um formato que pode ser exibido em um navegador da web ( ERDDAP™ em si mesmo
não exibe o vídeo, ele só fornece um link), e segundo, um csv
arquivo (ou um formato semelhante à tabela, como parquet) que contenham
metadados relevantes para os vídeos.

Ter um formato de vídeo apropriado é a chave. No exemplo abaixo do
vídeos originalmente eram o formato de uma velha janela que até mesmo navegadores do Windows
não poderia exibir fora da caixa, e necessário adicionar um plugin. Velho.
formatos muitas vezes são arquivos muito grandes. A aplicação Handbrake pode ser
usado para converter os vídeos, o que foi feito.

## Configuração

O primeiro passo é criar um conjunto de dados “EDDTableFromFileNames” para que
há uma URL associada a cada vídeo que é acessível através
o ERDDAP™ comando “arquivos”. Enquanto um exemplo deste conjunto de dados é dado
abaixo, a melhor maneira de gerar este xml para seus vídeos é usar
Gerar conjuntos de dados Xml.sh e apontá-lo para o diretório com os vídeos (em
general a melhor maneira de adicionar novos conjuntos de dados para ERDDAP™ é usar
Gerar conjuntos de dados Xml.sh que vai te tirar a maior parte do caminho lá, e
então edite isso para o resultado final).

O segundo passo é criar uma tabela EDDFromAsciiFiles (ou similar) 
dataset que contém os metadados que devem ser usados no vídeo
Procura. Um exemplo é dado abaixo. Para isto trabalhar deve haver um
coluna que contém um link para o vídeo, no exemplo é o
variável “videoFile”. Esta variável pode apenas conter a URL completa
criado no primeiro passo, mas então o arquivo não é portátil, como se
você quer testar em uma máquina e então realmente servir em outra
Máquina.

Você pode fazer o arquivo portátil, fazendo a entrada em “videoFile” para ser
o caminho relativo para o vídeo, no exemplo abaixo de um tal caminho é:

2010_Piggy_Bank_audio_stripped/0001_20100627_144631.mp4

Então você pode adicionar o atributo “fileAccessBaseUrl” que acrescenta que
URL para o caminho na coluna. No exemplo:

```
<att name="fileAccessBaseUrl"><https://coastwatch.pfeg.noaa.gov/erddap/files/fed_HAGE_VisualSurvey/></att>
```

Então, se você alternar servidores, você só precisa alterar esse atributo em
o snippet xml, você não precisa editar o arquivo csv.

Uma vez feito, você tem uma lista dos vídeos acessáveis como arquivos:

 [ https://coastwatch.pfeg.noaa.gov/erddap/files/fed_HAGE_VisualSurvey/ ](https://coastwatch.pfeg.noaa.gov/erddap/files/fed_HAGE_VisualSurvey/)
