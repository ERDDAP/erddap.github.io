# 搜尋和顯示影像 ERDDAP™ 

此文件顯示如何顯示影片 ERDDAP™ ,以及
建立數據集,提供搜尋影片中繼資料到
找到符合選擇搜尋條件的影片 。

首先, 影片是:
格式可以顯示在網頁瀏覽器中( Name ERDDAP™ 本身
不顯示影片, 它只提供連結),
文件 (或類似的表格格式, 如 parquet) 包含
和影片相關的中繼資料 。

使用适当的影像格式是關鍵 。 如下示例:
影片原本是舊的視窗格式,甚至連Windows瀏覽器也一樣
無法在框中顯示, 需要新增插件 。 老
格式通常是非常大的文件。 手剎應用程式可以是
轉換影片,

## 設定

第一步是建立「 EDDTable FileNames」 数据集, 以便
每段影片都有網址,
该 ERDDAP™ “文件”命令。 提供此數據集的示例
下方,您的影片產生此 xml 的最佳方式是使用
產生達塔斯 Xml.sh 用影片指向目錄
一般是新增數據集的最佳方式 ERDDAP™ 要使用
產生達塔斯 Xml.sh 它會帶你去大部份的路,而且
然後為最後結果編輯它。

第二步是建立 EDD Table From AsciiFiles 的 EDD Table 。 (或类似) 
包含影像中要使用的中繼資料集
搜索。 下面是一个例子。 要成功,一定有
列中包含有影片的連結,在示例中它是
變數“ videoFile ” 。 此變數只能包含完整 URL
在第一步建立, 但此檔案不是可移植的, 例如
你想在一臺機器上做測試 然后在另一台機器上做實驗
机器

您可以將「 videoFile」 中的項目做成可移植檔案
影片的相對路徑,

2010_Piggy_Bank_audio_stript/0001_20100627_144631.mp4

然後您可以加入「 fileAccessBaseUrl 」 屬性, 此屬性會附加
列中路徑的網址 。 例如:

```
<att name="fileAccessBaseUrl"><https://coastwatch.pfeg.noaa.gov/erddap/files/fed_HAGE_VisualSurvey/></att>
```

Xml 片段, 您不需要編輯 csv 檔案 。
