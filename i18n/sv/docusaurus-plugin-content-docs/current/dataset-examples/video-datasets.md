# Söka efter och Visa videor i ERDDAP™ 

Detta dokument visar hur man visar videor i ERDDAP™ samt
skapa en dataset som ger för att söka video metadata för att
Hitta de videoklipp som uppfyller de valda sökkriterierna.

De viktigaste komponenterna som behövs för att ställa in detta är först, videor som
finns i ett format som kan visas i en webbläsare (webbläsare) ERDDAP™ sig själv
Visa inte videon, det ger bara en länk), och andra, en csv
fil (eller ett liknande tabellbaserat format som parkett) som innehåller
metadata som är relevanta för videorna.

Att ha ett lämpligt videoformat är nyckeln. I exemplet nedanför
videor ursprungligen var ett gammalt fönster format som även Windows-webbläsare
kunde inte visa ur lådan och krävde att lägga till ett plugin. Gammal gammal
format är ofta mycket stora filer. Handbromsapplikationen kan vara
används för att konvertera videor, vilket är vad som gjordes.

## Setup

Det första steget är att skapa en "EDDTableFromFileNames" dataset så att
Det finns en URL associerad med varje video som är tillgänglig via
och ERDDAP™ "files" kommandot. Medan ett exempel på denna datamängd ges
Det bästa sättet att generera denna xml för dina videor är att använda
GenerateDatasets Xml.sh och peka det på katalogen med videorna (i
Allmänt det bästa sättet att lägga till nya datamängder ERDDAP™ är att använda
GenerateDatasets Xml.sh som kommer att få dig mest av vägen dit, och
sedan redigera det för slutresultatet).

Det andra steget är att skapa en EDDTableFromAsciiFiles (eller liknande) 
dataset som innehåller metadata som ska användas i videon
Sök. Ett exempel ges nedan. För att detta ska fungera måste det finnas en
kolumn som innehåller en länk till videon, i exemplet är det
Variabel "videoFile". Denna variabel kan bara innehålla hela URL
skapas i första steget, men sedan är filen inte bärbar, till exempel om
du vill testa på en maskin och sedan faktiskt tjäna på en annan
maskin.

Du kan göra filen bärbar genom att göra posten i "videoFile" för att vara
den relativa vägen till videon, i exemplet under en sådan väg är:

2010_Piggy_Bank_audio_stripped/0001_20100627_144631.mp4

Då kan du lägga till "fileAccessBaseUrl"-attributet som lägger till det
URL till vägen i kolumnen. I exemplet:

```
<att name="fileAccessBaseUrl"><https://coastwatch.pfeg.noaa.gov/erddap/files/fed_HAGE_VisualSurvey/></att>
```

Om du byter servrar behöver du bara ändra det ena attributet i
xml snippet, du behöver inte redigera csv-filen.
