Přispívání k ERDDAP™

Vykazování emise​

Neotvírejte problém GitHub, pokud je chyba bezpečnostní zranitelnost , a místo toho odkazovat na naše bezpečnostní politika . Všechny citlivé problémy budou odstraněny z veřejného sledovače.

Ujistěte se, že chyba nebyla již nahlášena vyhledáváním na GitHubu pod Otázky .

Pokud máte požadavek na funkci, aby se Github Diskuse (Více informací naleznete níže) .

Pokud nejste schopni najít otevřený problém řešení problému, Otevři novou. . Ujistěte se, že obsahuje název a jasný popis, co nejvíce relevantní informace, a pokyny pro reprodukci otázky nebo ERDDAP™ server demonstrující očekávané chování, které se neděje.

Řešení problému​

Otevřít novou GitHub pull žádost (PR) s náplastí.

Zajistit, aby PR popis jasně popisuje problém a řešení. Uveďte případně příslušné číslo emise.

Před odesláním si přečtěte Průvodce programátorem průvodce vědět více o kódování úmluv.

Přidání nové funkce​

Navrhněte změnu ERDDAP™ GitHub diskuse a začít psát kód. Před vytvořením GitHub diskuse, by mohlo být užitečné přečíst pokyny .

Neotvírejte problém na GitHub, dokud jste neshromažďovali pozitivní zpětnou vazbu o změně. Problémy s GitHub jsou primárně určeny pro hlášení chyb a opravy.
