Bidra till ERDDAP™

Rapportera ett problem​

Öppna inte ett GitHub-problem om felet är en säkerhetssårbarhet och istället hänvisa till vår Säkerhetspolicy . Alla känsliga frågor kommer att tas bort från den offentliga trackern.

Se till att felet inte redan rapporterades genom att söka på GitHub under Frågor .

Om du har en funktionsförfrågan gör en Github Discussion (se nedan för mer information) .

Om du inte kan hitta ett öppet problem med problemet, öppna en ny . Var noga med att inkludera en titel och tydlig beskrivning, så mycket relevant information som möjligt och anvisningar för att reproducera problemet eller en ERDDAP™ Server som visar det förväntade beteendet som inte förekommer.

Fastställande av en Problem​

Öppna en ny GitHub pull request (PR) med patchen.

Se till att PR-beskrivningen tydligt beskriver problemet och lösningen. Inkludera det relevanta emissionsnumret i förekommande fall.

Innan du skickar, läs gärna Programguide guide för att veta mer om kodningskonventioner.

Lägga till en ny funktion​

Föreslå din förändring i ERDDAP™ GitHub diskussioner och börja skriva kod. Innan du skapar GitHub-diskussionen kan det vara bra att läsa detta. vägledning .

Öppna inte ett problem på GitHub förrän du har samlat in positiv feedback om förändringen. GitHub-frågor är främst avsedda för buggrapporter och fixar.
